# Введение к Практике 5. Реализация Random Forest

## Цель практики

Самостоятельно **реализовать класс Random Forest** на основе решающих деревьев из sklearn. Можно выбрать либо **регрессию**, либо **классификацию** (в случае классификации — с поддержкой многоклассовой). После реализации нужно протестировать алгоритм на **любом подходящем датасете**.

## Основные этапы алгоритма

1. **Формирование обучающей выборки для каждого дерева:** случайный выбор объектов и признаков (bootstrap и/или subsample признаков).
2. **Обучение решающих деревьев:** для каждой такой выборки строится одно дерево (минимизация ошибки предсказания целевой переменной).
3. Шаги 1–2 повторяются для `n_estimators` деревьев.
4. **Агрегирование:** для каждого объекта собираются предсказания всех деревьев; итоговый ответ — среднее (регрессия) или голосование (классификация).
5. **Оценка качества** на тестовых данных (и при необходимости донастройка гиперпараметров).

## Ограничения и разрешённые средства

- **Разрешено:** использовать `sklearn.tree.DecisionTreeClassifier` и `sklearn.tree.DecisionTreeRegressor` как базовые модели.
- **Нельзя:** использовать готовые `sklearn.ensemble.BaggingClassifier` и `sklearn.ensemble.BaggingRegressor` — ансамбль нужно реализовать самим.
- **Обязательно:** сдача включает именно ваш код реализации; без него задание не засчитывается.

## Что нужно сделать

- Реализовать класс (регрессия или классификация) с описанной логикой Random Forest.
- Выбрать датасет и провести эксперименты: обучение, оценка качества, при необходимости сравнение с `sklearn.ensemble.RandomForestClassifier/Regressor` (по желанию).

## Рекомендации

- Чётко разделите: bootstrap/subsample признаков, обучение деревьев, агрегирование предсказаний.
- Для многоклассовой классификации учтите, что каждое дерево и голосование должны корректно работать с несколькими классам.
- Оформите выводы и, при необходимости, краткий отчёт по экспериментам.
